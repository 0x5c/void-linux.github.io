"We are the void, we use linux"

Void Linux is a lightweight and simple distribution optimized for the x86 (32bit) and x86_64 (64bit) CPU architectures. Currently there are ~2000 packages available in public repositories to build and install from.

Features:

  • It's not a fork or customized/modified version of any current distribution, it has been created from scratch.
  • It's a rolling release with continuos package updates.
  • The installed base system only provides required utilities for text console usage, adapt it to your needs!
  • Kernel images/modules are never removed, providing an always-bootable system.
  • Uses a modified initramfs-tools package from Debian to handle initial ramdisk images.
  • Uses a modified casper package from Ubuntu to boot from live images.
  • Uses a modified DKMS package to handle easy installation of Linux kernel modules.
  • Uses systemd as system and session manager without SysV init compatibility!
  • Uses an alternative and fast binary package system: XBPS (http://code.google.com/p/xbps)
  • The binary package builder (xbps-src) always builds packages in chroots, without requiring root privileges. Required dependencies can be resolved from local/remote repositories or directly from the source packages tree.
  • It's possible to build binary packages for Void Linux from your favourite linux distribution.

Source repositories

  • xbps-src (binary package builder) -- https://github.com/xtraeme/xbps-src

Download installable live images

NOTE: 256MB of RAM is required to boot and 500MB to install the base system.
NOTE: installation from network requires an ethernet device, manual configuration is required for WIFI.

  • For x86 compatible CPUs: http://xbps.goodluckwith.us/live
  • For x86_64 compatible CPUs: http://xbps.nopcode.org/live

Make sure to check the image integrity by matching the SHA256 hash in the .sha256 file.
