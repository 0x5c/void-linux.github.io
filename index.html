The Void Linux distribution

Void is a general purpose linux distribution, meant to be used by advanced users. Its package system allows you to quickly install, update and remove software; software is provided in binary packages or can be built directly from sources with the help of the XBPS source packages collection.

The spirit of Void is such that we keep things default, necessary, and minimal as possible (while trying not to leave features out).

Currently there are over 4000 optimized packages for the x86, x86_64, ARMv6 hard float (Raspberry Pi), and ARMv7 architectures; also there's support to build (natively or cross compiling) from sources any package easily that is available in the XBPS source packages collection.

Features

  • It is not a fork or customized / modified version of any current distribution, it has been created from scratch.
  • It is a rolling release with continuous package updates.
  • The installed base system only provides required utilities for text console usage, adapt it to your needs!
  • Kernel images / modules are never removed, providing an always-bootable system (x86 only).
  • runit as native init system/service supervision.
  • dracut to create initial ramdisk images.
  • dkms to simplify the management of third party linux kernel modules.
  • LibreSSL as default SSL/TLS implementation.
  • mdocml by default.
  • xbps as native package manager.
  • Multilib aware: compile and run 32bit code on 64bit platforms.
  • The XBPS source packages collection contains xbps-src, the binary package builder that builds packages in linux containers, without requiring root privileges. Required dependencies can be resolved from local/remote repositories or directly from the source packages tree.
  • Build your software from sources, without the need to use any binary package, on any linux distribution.
  • Supports installation on BIOS and UEFI 64bit systems (x86 only).

Download installable live images

Currently there are installable live images for the x86 and x86_64 architectures and there is support to make a local installation (with the included packages) or a network installation (packages are downloaded from official repository).

  • Void live with base-system (x86_64)
    Requirements: EM64T CPU, 96MB RAM, 350MB disk, Ethernet / WiFi for network installation.
  • Void live with base-system (i686)
    Requirements: Pentium 4 CPU (SSE2), 96MB RAM, 350MB disk, Ethernet / WiFi for network installation.

Log in as anon/root, password voidlinux.

To start the installer just execute the void-installer utility with enough permissions (i.e sudo).

Additional images with flavours are also available:

  • Void live with Enlightenment (x86_64)
  • Void live with Enlightenment (i686)
  • Void live with Mate (x86_64)
  • Void live with Mate (i686)
  • Void live with XFCE (x86_64)
  • Void live with XFCE (i686)

Download ready to boot images for ARM

The current list of supported ARM platforms is:
  • BeagleBone/BeagleBone Black (ARMv7, hard float)
  • Cubieboard2 (ARMv7, hard float)
  • Odroid U2/U3 (ARMv7, hard float)
  • Raspberry Pi (ARMv6, hard float)

Install Void by using a prepared image or a rootfs tarball that can be written / unpacked onto the SD card. This method allows you to have a system ready to boot / use, once it's written / unpacked onto the target device.

  • BeagleBone/BeagleBone Black image for 2GB SD card (EXT4): void-beaglebone-latest.img.xz
  • BeagleBone/BeagleBone Black rootfs: void-beaglebone-rootfs-latest.tar.xz
  • Please read the documentation to prepare the SD card here.
  • Cubieboard2 image for 2GB SD card (EXT4): void-cubieboard2-latest.img.xz
  • Cubieboard2 rootfs: void-cubieboard2-rootfs-latest.tar.xz
  • Please read the documentation to prepare the SD card here.
  • Odroid U2/U3 image for 2GB SD card (EXT4): void-odroid-u2-latest.img.xz
  • Odroid U2/U3 rootfs: void-odroid-u2-rootfs-latest.tar.xz
  • Please read the documentation to prepare the SD card here.
  • Raspberry Pi image for 2GB SD card (EXT4): void-rpi-latest.img.xz
  • Raspberry Pi image for 2GB SD card (F2FS): void-rpi-f2fs-latest.img.xz
  • Raspberry Pi rootfs: void-rpi-rootfs-latest.tar.xz
  • Please read the documentation to prepare the SD card here.
