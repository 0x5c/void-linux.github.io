"We are the void, we use linux"
Void Linux is a lightweight and simple distribution fully built from the source package tree available for the xbps-src build system and that can be customized in many ways through the use of virtual packages. Currently there are ~2000 packages available in repositories to build and install from. There is support for i686 and x86_64 architectures, more may be added in a future (i.e. ARM) but it's not a high priority.

Features:

  • It's not a fork or customized version of any distro, it has been created from scratch making it an extremely lightweight distro. Base system packages can be customized with your preferred virtual packages.
  • A rolling release with continuos package updates.
  • Uses the latest stable Linux kernel. The development snapshot from kernel.org can also be installed in parallel.
  • Kernel images/modules are never removed, providing an always-bootable system.
  • Uses a modified initramfs-tools package from Debian to handle initial ramdisk images.
  • Uses a modified casper package from Ubuntu to boot from live images.
  • Uses systemd as system and session manager, no SysV compatibility!
  • Uses a simplified DKMS package to handle easy installation of Linux kernel modules.
  • Uses its own binary package system: XBPS -- A faster/leaner alternative to dpkg/apt, RPM, Pacman, etc.
  • You don't need wrappers of wrappers for the XBPS utils to rebuild/reinstall packages. The provided tools are all that you need.
  • Uses a BSD port-like system to build binary packages in chroot jails without requiring root permissions. This allows building packages as a regular user without interfering with your installed system!

Download installable live images

  • For i686 compatible CPUs: http://xbps.goodluckwith.us/live
  • For x86_64 compatible CPUs: http://xbps.nopcode.org/live
Make sure to check the image integrity by matching the SHA256 hash in the .sha256 file.
