The Goods

Speed, reliability, and flexibility. That's the battle-cry of today's disgruntled computer geeks. This is what people want with a Linux distribution, and this is what Void Linux provides.

With lightning-fast and useful tools such as the XBPS package manager, and systemd (among others), this distribution clearly has all your base, and they rightly belong to us.

Void Linux was assembled manually by an infinite pack of rabid flesh-eating monkeys on a desert island back in 1904 (we are 2012-compliant).

Actually, it's a small project run by a couple of geeks in their spare time who don't get paid. We hang out on irc at #xbps on freenode.net quite a bit. Occasionally, idiots join the channel that just don't get it, and they end up leaving because they couldn't see how they could turn this awesome project into a freakin' Canonical clone.

We just hope that you like it, are able to contribute to it as your needs require; and, that if we don't like your contributions, they will be bitterly criticized until you fix it. No Warranties. :)

The spirit of Void is such that we keep things default, necessary, and minimal as possible (while trying not to leaving features out).

Features

  • It is not a fork or customized/modified version of any current distribution, it has been created from scratch.
  • It is a rolling release with continuous package updates.
  • The installed base system only provides required utilities for text console usage, adapt it to your needs!
  • Kernel images/modules are never removed, providing an always-bootable system.
  • Uses dracut to handle initial ramdisk images.
  • Uses a modified DKMS package to handle easy installation of Linux kernel modules.
  • Uses systemd as system and session manager without SysV init compatibility. We use systemd tools to configure any aspect of Void Linux.
  • Uses an alternative and fast binary package system: XBPS (code.google.com/p/xbps)
  • The binary package builder (xbps-src) always builds packages in chroots, without requiring root privileges. Required dependencies can be resolved from local/remote repositories or directly from the source packages tree.
  • Realtime Package Building: packages are built in VMs as soon as the changes are pushed to the git source repository; we can see in realtime when a change is problematic. See http://void-builder.uni.me:8010.
  • It is possible to build binary packages for Void Linux from your favourite linux distribution.
  • Supports installation on BIOS and EFI systems.
